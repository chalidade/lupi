# About

Lupi is a Universal API that builds over Laravel microservices (lumen). The API on this Lupi is reusable, and it seems like a starter API that you can modify as you want. Especially if you are familiar with Laravel, it's easy to read and modified Lupi. Lupi will cover almost all about the backend process and make the front-end more the fastest on the development process. The API in Lupi is dynamic, you can change parameters as you want and get the result. Let's try Lupi and share your experience

Foundation of Lupi is Lumen, it means all of Laravel syntax and function can be used to modify this API. You can using all of Laravel or Lumen documentation over Lupi. Absolutely you must install composer and other requirements to running laravel well on your computer before using Lupi. Lumen choosing for a foundation that because lumen is lightwell than laravel and if you need other packages you can install as you need.

# Installation

Installation Lupi is very easy, It seems like you download a template that you can use or modify as you want. The thing that you need to know is Lupi create based on Laravel microservices (Lumen) all of the function and package referrers to Lumen as a foundation to build Lupi, include service requirement and installation process.

Server Requirement

As shown on Lumen documentation that Laravel framework has a few system requirement. All of these requirement are satisfied by the Larvel Homestead virtual machine, so it's highly recommended that you use Homestead as your local Laravel development environment.

However, if you are not using Homestead, you will need to make sure your server have the following requirements :

  • Install Xampp / Local Server
  • PHP >= 7.1.3
  • Open SSL PHP Extention
  • PDO PHP Extention
  • Mbstring PHP Extention


Installing Composer

Lumen utilized Composer to manage it's dependencies. So Lupi need this too. Before using Lupi, make sure you have Composer installed on your server. If you need detail about this, you can visit How to Install Composer.


Install Lupi

It's easy to install Lupi in your machine, just following this step :

  • First, open Lupi repository
  • Choose Clone / Download
  • If you choose Download Zip, after downloading open your htdoc folder if you're using Xampp or /var/www/html/ if you're using linux
  • Then Extract and wait until extraction process done
  • Open command prompt and open that directory
  • Then, type composer install -> Enter
  • After installation process, open your browser then open this project in you localhost

If you got error like Fatal error: require_once(): Failed opening required, like this.

Warning: require_once(D:\xampp\htdocs\lupi\bootstrap/../vendor/autoload.php): failed to open stream: No such file or directory in ...
Fatal error: require_once(): Failed opening required ‘D:\xampp\htdocs\lupi\bootstrap/../vendor/autoload’ (include_path=’D:\xampp\php\PEAR’) in ...

It mean, your composer install failed. if you have problem on installation process you can send me email chalidade@gmail.com

If you want to adding new database configuration in Lupi, it's simple. Open config folder, then edit database.php. Copy exampleMysql then change value as your database configuration.

'exampleMysql' => [
'driver' => 'mysql',
'host' => env('DB_HOST', 'localhost'),
'port' => env('DB_PORT', 3306),
'database' => env('DB_DATABASE', 'your_database'),
'username' => env('DB_USERNAME', 'your_username'),
'password' => env('DB_PASSWORD', 'your_pass'),
'charset' => env('DB_CHARSET', 'utf8'),
'collation' => env('DB_COLLATION', 'utf8_unicode_ci'),
'prefix' => env('DB_PREFIX', ''),
'timezone' => env('DB_TIMEZONE', '+00:00'),
'strict' => env('DB_STRICT_MODE', false),
],

# Oracle Database Configuration
'exampleOracle' => [ 'driver' => 'oracle',
'tns' => env('DB_TNS', ''),
'host' => env('DB_HOST', 'localhost'),
'port' => env('DB_PORT', '1521'),

'username' => env('DB_USERNAME', 'your_username'),
'password' => env('DB_PASSWORD', 'your_password'),

Explanation :
  1. exampleMysql is name of your database configuration, you will call it later in controller or json format
  2. Drivers is database type that we use
  3. Host is your server, you can fill with domain or IP. If you're running in local server you can use localhost
  4. Database is name of your database
  5. Username is your username when you want to access database
  6. Password is your password when you want to access database

# Start Limit

Start and limit in index funtion using for pagination or get limited data as you needs. Start is use to get data from index x. then limit use to indentified how many data that we want to get. As example, if we want to get data in table TX_HDR_BOOK from index 0 and we want get only 2 data. So we use json format like this.

{
"action" : "list",
"db" : "library",
"table" : "TX_HDR_USER",
"start" : "0"
"limit" : "2"
}

That's parameter will generate SQL Script like below

SELECT * FROM `TX_HDR_BOOK` LIMIT 0,2

Absolutly you can combine start and limit with any parameter from index function to get result as you want.

# Where Statement

Where Statment is a parameter that often we will use. This parameter can be used to filtering data or get data with a keyword. Where statements can be used with other parameter include join parameter. There are few where statement that explain below


# Basic Where

This is basic where template and how where statement explain

{
"action" : "list",
"db" : "your_database_config",
"table" : "your_table",
"where" : [
["field1", "=", "value"],
["field2", ">", "value"],
.....
.....
]
}

Basic where statement will generate AND condition in every field and value. In that case, that will generate an SQL code like below

SELECT * FROM `your_table` WHERE `field1` = 'value' AND `field2` > 'value' AND `field3` = 'value' AND ......... AND .........

Every new condition will generate AND condition on Basic where. Format of statements is patent, ['field', 'operator', 'value'] as long as you don't modified the main of LUPI controller. All of operator that you apply on SQL code, you can use in LUPI too.


# Example 1

{
"action" : "list",
"db" : "library",
"table" : "TX_HDR_BOOK",
"where" : [
["BOOK_CATEGORY", "=", "FICTION"]
],
"limit" : "2"
}

Those parameter will generate an SQL Query like below

SELECT * FROM `TX_HDR_BOOK` WHERE `BOOK_CATEGORY` = 'FICTION' LIMIT 2

# Example 2

{
"action" : "list",
"db" : "library",
"table" : "TX_HDR_BOOK",
"where" : [
["BOOK_CATEGORY", "=", "FICTION"],

]
}

Like SQL Query, you can use 'like' operator as condition, Those parameter will generate an SQL Query like below

SELECT * FROM `TX_HDR_BOOK` WHERE `BOOK_CATEGORY` = 'FICTION' AND `BOOK_TITLE` like '%farmer%'

# Where In

Where In use to get data in array value that we declare. As example we want to show data book on page 1,3,5,6 so we must use Where In for this condition. Where In Format show in format below

The IN operator allows you to specify multiple values in a WHERE clause. In that case, that will generate an SQL code like below

SELECT * FROM `your_table` WHERE `field1` IN ("value1","value2","value3",...,....)

# Joining Table

A join clause is used to combine rows from two or more tables, based on related column between them. There are two type join in LUPI, Left Join and Inner Join that will explain on below

# Inner Join

Inner Join will returns data that have matching values in both tables,If the values of one of the data do not match then the data will not be displayed, while the same data will still be displayed. This is basic where template and how where statement explain

{
"action" : "list",
"db" : "your_database_config",
"table" : "your_table",
"innerJoin" : [
{
"table" : "your_joining_table",
"field1" : "your_table.your_field_on_table",
"field2" : "your_joining_table.your_field_on_table",
},
{
"table" : "your_joining_table AS C",
"field1" : "C.your_field_on_table1",
"field2" : "your_table.your_field_on_table2",
}
]
}

We can joining more than two table by adding group of parameter ("table", "field1", "field2"). Table on join parameter mean your joining table, then field1 / field2 on join parameter mean your related field between table A and table B. You can use alias for your table to declare easier. For That Sample, we will generate an SQL Script like below

SELECT * FROM `your_table`
INNER JOIN `your_joining_table` ON `your_table`.your_field_on_table1 = `your_joining_table`.your_field_on_table
INNER JOIN `your_joining_table` AS C ON C.your_field_on_table1 = `your_table`.your_field_on_table2

# Example 1

{
"action" : "list",
"db" : "library",
"table" : "TX_HDR_BOOK as A",
"innerJoin" : [
{
"table" : "TX_DTL_BOOK as B",
"field1" : "B.DTL_HDR_ID",
"field2" : "A.BOOK_ID",
}
],
"start" : "0",
"limit" : "10"
}

That JSON will generate SQL Script like below

SELECT * FROM `TX_HDR_BOOK` as A INNER JOIN `TX_DTL_BOOK` as B ON A.BOOK_ID = B.DTL_HDR_ID LIMIT 0,10


# Left Join

The Left Join return all records from table1, and the matched record from the table2. The result is NULL from the right side where there is no data match. This is basic where template and how where statement explain

{
"action" : "list",
"db" : "your_database_config",
"table" : "your_table",
"leftJoin" : [
{
"table" : "your_joining_table",
"field1" : "your_table.your_field_on_table",
"field2" : "your_joining_table.your_field_on_table",
},
{
"table" : "your_joining_table AS C",
"field1" : "C.your_field_on_table1",
"field2" : "your_table.your_field_on_table2",
}
]
}
SELECT * FROM `your_table`
LEFT JOIN `your_joining_table` ON `your_table`.your_field_on_table1 = `your_joining_table`.your_field_on_table
LEFT JOIN `your_joining_table` AS C ON C.your_field_on_table1 = `your_table`.your_field_on_table2

# Example 1

{
"action" : "list",
"db" : "library",
"table" : "TX_HDR_BOOK as A",
"leftJoin" : [
{
"table" : "TX_DTL_BOOK as B",

}

The SELECT statement is used to select fields that we want to showing. We can choose what we want to show. example if we have table TX_HDR_BOOK that have fields BOOK_NO, BOOK_TITLE, BOOK_PAGE, BOOK_YEAR and we just want to show BOOK_TITLE and BOOK_YEAR so we can use select parameter

# Example 1

{
"action" : "list",
"db" : "library",
"table" : "TX_HDR_BOOK",
"selected" : ["BOOK_TITLE", "BOOK_YEAR"],
"start" : "0",
"limit" : "10"
}

That JSON will generate SQL Script like below

SELECT `BOOK_TITLE`, `BOOK_YEAR` FROM `TX_HDR_BOOK` LIMIT 0,10

Yes, Absolutly you can combine with other paramater like join or where statement


# Example 2

{
"action" : "list",
"db" : "library",
"table" : "TX_HDR_BOOK as A",
"leftJoin" : [
{
"table" : "TX_DTL_BOOK as B",
"field1" : "B.DTL_HDR_ID",

"A.BOOK_TITLE as TITLE",

"B.DTL_DATE_PUBLISH as PUBLISH"

}

We can aliases field as we want by adding as like above. That JSON will generate SQL Script like below

SELECT A.`BOOK_TITLE` as TITLE, B.`DTL_AUTHOR` as AUTHOR, B.`DTL_DATE_PUBLISH` as PUBLISH FROM `TX_HDR_BOOK` as A
LEFT JOIN `TX_DTL_BOOK` as B ON A.BOOK_ID = B.DTL_HDR_ID LIMIT 0,10


# Group Data By

The Group By parameter in Lupi usually use to group rows that have same value into sumary rows. An example you want to find the number of book that publish each year. The Group By parameter is often used with agregate function (COUNT, MAX, MIN, SUM, AVG) to group the result-set by one or more columns. Keep it mind that you need selected parameter to using group by in Lupi

# Example 1

{
"action" : "list",
"db" : "library",
"table" : "TX_HDR_BOOK",
"selected" : ["BOOK_AUTHOR", "BOOK_YEAR"],
"groupBy" : "BOOK_YEAR",
"start" : "0",
"limit" : "10"
}

That JSON will generate SQL Script like below

SELECT `BOOK_AUTHOR`, `BOOK_YEAR` FROM `TX_HDR_BOOK` GROUP BY `BOOK_YEAR` LIMIT 0,10

If you want to count how many book of author you can use combination of groupBy and raw select like below.


# Example 2

{
"action" : "list",
"db" : "library",
"table" : "TX_HDR_BOOK",
"raw" : {
"selected" : "BOOK_AUTHOR, count(BOOK_AUTHOR) as BOOK_TOTAL"
},
"groupBy" : "BOOK_AUTHOR",
"start" : "0",
"limit" : "10"
}

Raw parameter lets us use agregate funtion like count, sum, avg, and etc in select funtion. it mean we can use sql raw code in Lupi. which we will explain in another menu

SELECT `BOOK_AUTHOR`, count(`BOOK_AUTHOR`) as BOOK_TOTAL from TX_HDR_BOOK GROUP BY `BOOK_AUTHOR`


# Order Data By

In every condition we need to sort data by ascending or descending. The Order By parameter in Lupi is used to sort the result-set in ascending (ASC) or descending (DESC). In default Lupi will sort data by ascending if you need to sort the record to descending use DESC keyword.

# Example 1

{
"action" : "list",
"db" : "library",
"table" : "TX_HDR_BOOK",
"selected" : ["BOOK_AUTHOR", "BOOK_YEAR"],
"orderBy" : ["BOOK_YEAR", "DESC"],
"start" : "0",
"limit" : "10"
}

That JSON will generate SQL Script like below

SELECT `BOOK_AUTHOR`, `BOOK_YEAR` FROM `TX_HDR_BOOK` ORDER BY `BOOK_YEAR` DESC LIMIT 0,10

Absolutly we can combine with other parameter like join and where condition


# Example 2

{
"action" : "list",
"db" : "library",
"table" : "TX_HDR_BOOK as A",
"leftJoin" : [
{
"table" : "TX_DTL_BOOK as B",
"field1" : "B.DTL_HDR_ID",
"field2" : "A.BOOK_ID",
}
],
"orderBy" : ["A.BOOK_ID", "DESC"],
"start" : "0",
"limit" : "10"
}

The code will generate an SQL Code like below

SELECT * FROM `TX_HDR_BOOK` as A LEFT JOIN `TX_DTL_BOOK` as B ON A.BOOK_ID = B.DTL_HDR_ID ORDER BY A.BOOK_ID DESC LIMIT 0,10


# Raw Parameter

The raw parameter mean you can use an complex raw SQL script on this parameter if all function on lupi cannot cover your needs. Because some parameter in Lupi have fix format, you must follow it or you will get error. An example like selected parameter that if you adding aggregate you will get error. So You need to use this raw parameter.

# Example 1 | Incorrect
{
"action" : "list",
"db" : "library",
"table" : "TX_HDR_BOOK",
"selected" : ["BOOK_AUTHOR, "count(BOOK_AUTHOR) as BOOK_TOTAL"],
"groupBy" : "BOOK_AUTHOR",
"start" : "0",
"limit" : "10"
}

Wrong JSON, aggregate function in selected parameter will get an error because it read as field, so count(BOOK_AUTHOR) as BOOK_TOTAL read as field. if you don't have field named count(BOOK_AUTHOR) as BOOK_TOTAL you will get error. In query you will hit sql script like below

SELECT `BOOK_AUTHOR`, `count(BOOK_AUTHOR) as BOOK_TOTAL` from TX_HDR_BOOK GROUP BY `BOOK_AUTHOR`


# Example 1 | Correct
{
"action" : "list",
"db" : "library",
"table" : "TX_HDR_BOOK",
"raw" : {
"selected" : "BOOK_AUTHOR, count(BOOK_AUTHOR) as BOOK_TOTAL"
},
"groupBy" : "BOOK_AUTHOR",
"start" : "0",
"limit" : "10"
}

Correct JSON, different by selected in raw parameter, that count(BOOK_AUTHOR) as BOOK_TOTAL read as raw SQL Query, so it will give you data count of BOOK_AUTHOR as BOOK_TOTAL because count read as aggregate function not field

SELECT `BOOK_AUTHOR`, count(`BOOK_AUTHOR`) as BOOK_TOTAL from TX_HDR_BOOK GROUP BY `BOOK_AUTHOR`


# View Header Detail

If you separate data in header and detail, you can show them in action viewheaderdetail. This function is powerfull you can show data in one JSON from two or more table that have foreign key. An Example, I have two table : TX_HDR_USER and TX_DTL_USER. In TX_HDR_USER i choose USER_ID as primary key. Then in TX_DTL_USER i choose DTL_ID as primary key and DTL_HDR_ID as foreign key or ID that link into header data. Format view header detail you can see below


{
"action" : "viewheaderdetail",
"data" : [
"HEADER"
"DETAIL1"
"DETAIL2"
....
....
],
"HEADER" : {
"DB" : "your_database_config",
"TABLE" : "your_table",
"PK" : ["primary_key", "header_id"]
}
"DETAIL1" : {
"DB" : "your_database_config",
"TABLE" : "your_table",
"FK" : ["primary_key", "header_primary_key"]
},
"DETAIL2" : {
"DB" : "your_database_config",
"TABLE" : "your_table",
"FK" : ["primary_key", "header_primary_key"]
},
....
....
}
